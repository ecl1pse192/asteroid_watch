# Техническое задание на проект: NEO Observer

## 1. Цель проекта
Создание информационно-аналитической системы мониторинга сближающихся с Землей объектов (NEO). Сервис решает проблему сложного восприятия "сырых" научных данных NASA, предоставляя астрономам-любителям удобный инструмент для отслеживания потенциальных космических угроз и ведения личных списков наблюдения.

## 2. Роли пользователей
* **Гость:** Просмотр общего каталога астероидов на текущую неделю, доступ к статистическим графикам распределения по размерам и опасности.
* **Исследователь (Зарегистрированный пользователь):** Ведение личного «Списка отслеживания» (Watchlist), добавление персональных заметок к объектам, фильтрация списка по статусу "Потенциально опасный".

## 3. Модели данных
1. **Asteroid (Астероид)**
   * *Назначение:* Справочник уникальных небесных тел.
   * *Поля:* `nasa_id`, `name`, `absolute_magnitude`, `is_potentially_hazardous`, `nasa_jpl_url`.

2. **Flyby (Сближение)**
   * *Назначение:* Информация о конкретном пролете астероида мимо Земли.
   * *Поля:* `asteroid` (FK), `date`, `velocity_kmh`, `miss_distance_km`.

3. **Watchlist (Список отслеживания)**
   * *Назначение:* Персональные настройки мониторинга.
   * *Поля:* `user` (FK), `asteroid` (FK), `user_notes`, `added_at`.

## 4. Ключевой функционал
* **Мониторинг:** Система автоматически загружает данные о пролетах на текущую неделю и отображает их в виде таблицы.
* **Фильтрация:** Пользователь может скрыть безопасные объекты и оставить только те, что имеют статус "Потенциально опасный".
* **Личный контроль:** Пользователь добавляет интересующий астероид в Избранное и оставляет заметку.
* **Визуализация:** На главной странице строится график, показывающий соотношение безопасных и опасных объектов.

## 5. Внешние интеграции / Аналитика
* **Интеграция:** Использование внешнего **NASA NeoWs API** для автоматического получения актуальных данных о небесных телах.
* **Аналитика:** Использование библиотеки **Chart.js** на фронтенде для визуализации статистики.
