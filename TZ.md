# Техническое задание на проект: NEO Observer

## 1. Название и концепция
**Полное название:** Информационно-аналитическая система мониторинга сближающихся с Землей объектов (NEO) для астрономов-любителей.
**Концепция:** Веб-сервис агрегирует данные из API NASA, позволяя отслеживать астероиды, пролетающие в опасной близости от Земли.

## 2. Целевая аудитория и роли
* **Гость:** Просмотр списка астероидов и графиков статистики.
* **Исследователь (User):** Ведение личного «Списка наблюдения» (Watchlist).

## 3. Схема данных (Модели)
1. **Asteroid**: Хранит статичные данные (ID, название, размер, флаг опасности).
2. **Flyby**: Хранит данные о пролете (дата, скорость, дистанция). Связь с Asteroid.
3. **Watchlist**: Связывает пользователя и астероид. Поля: user, asteroid, notes.

## 4. Ключевой функционал
1. **Мониторинг:** Таблица пролетов на текущую неделю.
2. **Фильтрация:** Фильтр "Только опасные".
3. **Личный контроль:** Добавление в избранное с заметками.
4. **Аналитика:** График распределения астероидов по размерам.

## 5. Стек
Python 3.12, Django 5.0, SQLite, NASA NeoWs API, Bootstrap 5.
